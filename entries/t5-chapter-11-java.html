11.2 Superclasses & Subclasses

Key Point: Inheritance enables you to define a general class (a superclass) and later extend it to more specialized classes (subclasses).

Other terms used to define them:

  • Superclass: Parent or Base class
  • Subclass: Child, Extended or Derived class
A subclass inherits accessible data fields and methods from its superclass and may also add new data fields and methods. In Java, we use the term extends to define which superclass our subclass will inherit from.

Some important points regarding inheritance:

  • A subclass usually contains more information and methods than its superclass.
  • Private data fields in a superclass are only accesible through public accessors/mutators.
  • Inheritance is used to model the is-a relationship. However, not all is-a relationships should be modeled using inheritance. It's important to properly use abstraction.
  • Java does not allow multiple inheritance. A Java class may inherit directly from only one superclass.
